# NotebookLM Workflow Reference

## Use this reference when

- The source set is mixed or messy.
- You need to choose what belongs in NotebookLM.
- You need stable output templates.

## Triage checklist

For each candidate source, check:

1. What is it?
2. Who produced it?
3. When was it produced?
4. Is it a primary source or a derivative summary?
5. Does it overlap another source?
6. Should it be uploaded, summarized outside NotebookLM, or excluded?

## Suggested artifact templates

### `source-manifest.md`

```md
# Source Manifest

## Recommended upload order
1. ...
2. ...

## Sources
- **Source name** — type, date, why it matters
- **Source name** — type, date, why it matters
```

### `briefing.md`

```md
# Notebook Briefing

## Objective
- ...

## Scope
- ...

## Important entities
- ...

## Known gaps
- ...
```

### `question-set.md`

```md
# Suggested Questions

- What changed between ... and ...?
- Which sources support ...?
- Where do the sources disagree?
- What evidence is still missing?
```

## Quality bar

- Prefer original source documents over summaries.
- Keep chronology explicit when time matters.
- Make uncertainty visible.
- Avoid invented citations.
- Do not merge separate entities just because names look similar.
